# 📚 BOOK CLUB 대시보드

노션 없이 여러 독서모임이 초대 코드로 각자의 대시보드를 쓸 수 있는 사이트입니다. 지금 이 폴더를 열면 **데모 모드**로 바로 동작하고(가짜 예시 데이터), 아래 단계를 따라 Supabase를 연결하면 실제로 멤버들과 데이터를 공유하는 사이트가 됩니다.

## 지금 바로 미리보기

`index.html`을 더블클릭해서 열면 랜딩 화면이 뜹니다. 데모 모드에서는 "내 독서모임 찾기"에 초대 코드 `DEMO01`을 입력하면 예시 데이터가 채워진 모임으로 바로 들어가볼 수 있어요. (이 상태에서 입력한 내용은 새로고침하면 사라져요 — 아래 1~2단계를 마쳐야 실제로 저장됩니다.)

## 1. Supabase 프로젝트 만들기 (무료, 5분)

1. [supabase.com](https://supabase.com) 에서 가입 → **New project** 생성 (이름은 아무거나, 리전은 Northeast Asia/Seoul 추천)
2. 프로젝트가 만들어지면 왼쪽 메뉴 **SQL Editor** 클릭 → **New query**
3. 이 폴더의 `schema.sql` 파일 내용을 전체 복사해서 붙여넣고 **Run** 실행
   - 모임(clubs)/멤버/공지/책/참여현황/활동기록/좋아요/댓글/추천 테이블 + 통계용 뷰 + 접근 권한(RLS)까지 한 번에 만들어져요
4. 왼쪽 메뉴 **Settings → API** 클릭
   - **Project URL** 복사
   - **anon public** 키 복사 (secret 키 아님! anon 키만 사용)

## 2. 사이트에 연결하기

`js/config.js` 파일을 열어서 아래 두 줄을 방금 복사한 값으로 채워주세요.

```js
const CLUB_CONFIG = {
  SUPABASE_URL: "https://xxxxxxxx.supabase.co",
  SUPABASE_ANON_KEY: "eyJhbGciOi...",
  CLUB_TITLE: "BOOK CLUB"
};
```

저장하고 `index.html`을 다시 열면 화면 위 "데모 모드" 배너가 사라지고, 실제 Supabase에 데이터가 저장됩니다.

## 3. 모임 만들기 / 참여하기

이제 책이나 참여자를 Supabase에서 직접 넣을 필요 없이, 전부 사이트 화면에서 처리됩니다.

- **모임을 처음 시작하는 사람**: 랜딩 화면에서 "독서모임 만들기"에 모임 이름 + 내 이름 입력 → 6자리 초대 코드가 발급되고, 그 사람이 **방장**이 됩니다.
- **멤버로 합류하는 사람**: "내 독서모임 찾기"에 방장에게 받은 초대 코드 + 내 이름을 입력하면 바로 대시보드로 들어갑니다.
- 코드는 상단바의 모임 이름 옆 배지를 클릭하면 복사돼요.

## 4. 방장 전용 기능

방장(모임을 만든 사람)에게만 보이는 기능이 있어요.

- **책 등록 / 완료 처리**: "이번 독서모임 도서" 영역의 "+ 책 등록"으로 새 책 추가 (등록하면 이전 책은 자동으로 지난 독서모임으로 이동), "완료 처리"로 지금 책을 아카이브로 보내기
- **공지사항 편집**: 방장만 새 공지를 등록할 수 있어요
- **멤버 관리**: 상단바 "멤버 관리"에서 멤버 목록 확인, 강퇴 가능
- **모임 삭제**: 화면 맨 아래 "모임 삭제" — 책/활동기록/추천 등 모든 데이터가 함께 삭제됩니다 (되돌릴 수 없음)

> 참고: 로그인 시스템이 없는 가벼운 구조라 "강퇴"는 멤버 목록에서 이름을 지우는 정도예요. 초대 코드를 아는 사람은 다시 이름을 입력하고 들어올 수 있으니, 정말 민감한 모임이라면 코드 유출에 주의해주세요.

## 5. 멤버들과 공유하기 (배포)

혼자 로컬 파일로 봐도 되지만, 멤버들이 각자 접속하려면 인터넷에 올려야 해요. 둘 다 무료입니다.

### Netlify (가장 쉬움)
1. [app.netlify.com/drop](https://app.netlify.com/drop) 접속
2. 이 `book-club-app` 폴더를 통째로 화면에 드래그 앤 드롭
3. 몇 초 후 나오는 URL을 멤버들에게 공유하면 끝

### GitHub + Netlify/Vercel (수정이 잦다면 추천)
1. 이 폴더에서 터미널 열고 `git init` (아직 git 저장소가 아니라면)
2. GitHub에 새 빈 저장소 생성 후 `git remote add origin <URL>` → `git add -A` → `git commit -m "책 클럽 사이트"` → `git push -u origin main`
3. Netlify/Vercel에서 "Import from Git"으로 이 저장소 연결 (빌드 명령 없음, publish directory는 저장소 루트)
4. 이후 `git push`만 하면 사이트가 자동으로 갱신돼요

## 사용 방법 요약

- **랜딩**: 모임 만들기(방장이 됨, 초대 코드 발급) 또는 초대 코드로 찾기(멤버로 합류)
- **공지사항**: 상단 고정, 방장만 새 공지 등록 가능
- **이번 책**: 방장이 등록한 책 중 `status = ongoing`인 책이 자동으로 표시됨
- **참여 현황**: 표에서 바로 상태/별점/한줄평 수정, 평균 별점·참여 인원은 자동 계산
- **활동 기록**: 문장 수집 / 발제 / 독후감 / 토론 후 감상 탭 전환, 좋아요·정렬(발제는 좋아요순 추천)·댓글(발제용) 지원
- **추천 도서**: 하단에서 다음 책 추천 등록 → "추천하기"로 투표 → 추천 많은 순 정렬 → 방장이 "이 책으로 선정"
- **지난 독서모임**: 방장이 완료 처리한 책이 자동으로 아카이브에 이동, 클릭하면 상세 기록 열람
- **마이페이지**: 내 이름 기준으로 내가 쓴 글 모아보기

## 디자인

소진님이 주신 무드보드(머스터드 골드 + 딥 플럼 + 월넛 브라운, 세리프 로고 대신 전체 서체는 Pretendard 통일)를 반영해서 색상/폰트를 적용했어요. 색은 `style.css` 맨 위 `:root` 변수(`--gold`, `--plum`, `--walnut` 등)만 바꾸면 전체 톤이 한 번에 바뀝니다.

## 지금 범위에 포함하지 않은 것 (원 스펙의 "추후" 항목)

- **자동 알림/리마인드** (추천 종료 알림, 모임 리마인드 등): 정적 사이트라 서버가 없어서 자동 알림 발송은 불가능해요. 필요하면 나중에 카카오톡/문자 자동화(예: Make, Zapier)를 붙이는 방식으로 확장 가능합니다.
- 마이페이지는 간단 버전입니다 (내가 읽은 책 + 활동 목록). 나중에 프로필 사진, 뱃지 등으로 확장 가능.

## 주의할 점

- `anon` 키는 "누구나 읽고 쓸 수 있는" 공개 접근 권한으로 열려 있어요. 소규모 지인 모임 전제라 로그인 시스템 없이 이름 + 초대 코드만으로 구분하는 가벼운 구조입니다. 코드가 공개적으로 퍼지지 않게만 주의하고, 민감한 개인정보는 넣지 마세요.
- 나중에 진짜 로그인(이메일/비밀번호)이 필요해지면 Supabase Auth를 추가로 붙일 수 있어요 — 필요하면 말씀해주세요.
